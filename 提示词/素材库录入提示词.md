## 提取规则

从拆解结果中提取以下全部字段：

| 字段 | 从哪里提取 |
|:---|:---|
| **视频名称** | 用户提供，或从内容概括简短名称 |
| **视频类型** | 判断视频是「故事类」（靠叙事弧线驱动）还是「形式类」（靠视觉/听觉表现形式驱动） |
| **视频时长** | 视频反推的元数据 |
| **故事大纲** | 视频反推的完整剧情。角色命名规则：若能识别为常驻角色（见下方角色表），使用角色名；否则用身份或外观特征命名 |
| **开头标签** | 从固定标签中选择（见下方标签表），多个用逗号分隔 |
| **开头** | 前1-3秒的视觉钩子描述 |
| **视频标签** | 从固定标签中选择（见下方标签表），多个用逗号分隔 |
| **标签手法** | 视频靠什么手法达成标签效果的 |
| **机制名称** | 核心机制的简称/代号 |
| **机制链条** | 剥离表面元素后的底层运转逻辑 |
| **主角** | 按道德立场判定的正方主角（承受最大状态变化、观众情感绑定的角色）。使用常驻角色名或身份/特征 |
| **主角目标** | 主角想要达成什么 |
| **反派** | 与主角对立、制造核心冲突的角色。无反派则留空 |
| **反派目标** | 反派想要达成什么。无反派则留空 |

> **机制一致性规则**：母版及其所有子版必须共享同一个机制名称和机制链条。录入子版时，若发现核心机制可以进一步抽象，先询问用户是否需要同步更新母版的机制，确认后再统一修改。

| **改编标签** | 从固定标签中选择。若用户指定了母版视频必须填写；若为原创则留空 |
| **改编简介** | 从原视频改了什么、保留了什么。若为原创则留空 |
| **母版视频** | 若用户指定了库中某个视频是母版，填写该视频的数据库ID。若为原创则留空 |
| **场景** | 视频反推 →【全局设定】→ 场景 → 名称 + 场景功能 |
| **道具** | 视频反推 →【全局设定】→ 核心道具 → 名称 + 类型 + 作用 |
| **角色** | 视频反推 →【全局设定】→ 角色 → 名称（用常驻角色名或身份/特征命名） + 人设（叙事功能） + 特殊能力（超常能力，逗号分隔，无则留空） + 特殊状态（非常态形态/状态，逗号分隔，无则留空） |
| **元数据** | 视频标题、发布日期、链接、播放量、点赞量 → 从反推元数据提取 |
| **缩略图** | 从视频链接自动生成并下载到本地。录入后运行 `node backup_thumbs.js` 从 YouTube 下载缩略图保存到 `public/thumbs/`，数据库存储本地路径 `/thumbs/xxx.jpg` |
| **录入日期** | 用户提供，或用今天日期 |
| **脚本路径** | 若用户提供了脚本文件，填写相对路径 |

## 写入方式

调用素材库 API（运行在 `http://localhost:3456`）：

```bash
curl -X POST http://localhost:3456/api/videos \
  -H "Content-Type: application/json" \
  -d '{
    "name": "视频名称",
    "video_type": "故事类",
    "duration": "48",
    "summary": "故事大纲",
    "hook_tags": "异常行为, 冲突",
    "hook": "开头描述",
    "video_tags": "荒诞, 搞笑",
    "technique": "标签手法",
    "mechanism_name": "机制名称",
    "mechanism": "机制链条",
    "protagonist": "jinu",
    "protagonist_goal": "找出谁是假Rumi",
    "antagonist": "mira",
    "antagonist_goal": "伪装成Rumi不被识破",
    "adapt_tags": "换皮型改编",
    "adapt_brief": "改编简介",
    "source_video_id": "",
    "video_title": "原视频标题",
    "publish_date": "2026-01-15",
    "date": "2026-03-25",
    "video_link": "https://...",
    "thumb_url": "https://img.youtube.com/vi/VIDEO_ID/maxresdefault.jpg",
    "views": "8726165",
    "likes": "106657",
    "script_path": "scripts/xxx.md",
    "scenes": [
      {"name": "场景名称", "function": "场景功能"}
    ],
    "props": [
      {"name": "道具名称", "type": "类型", "function": "作用"}
    ],
    "characters": [
      {"name": "角色名称", "persona": "人设", "abilities": "喷火, 冰冻", "states": "变身绿巨人"}
    ]
  }'
```

写入成功后：

1. 告诉用户已录入，并简要列出录入了哪些内容
2. **自动执行视频备份**（下载视频并上传到阿里云 OSS）：

```bash
cd /Users/renzengfei/短视频/素材库 && node backup.js <新录入的video_id>
```

备份成功后，数据库会自动更新 `video_path`（OSS 备份链接）和 `thumb_url`（YouTube 缩略图链接）。

> ⚠️ 如果素材库服务器未启动，先运行：`cd /Users/renzengfei/短视频/素材库 && node server.js`
